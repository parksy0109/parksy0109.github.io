- 언어: 기본은 한국어
- 톤: 차분하고 선명한 실무형
- 스타일: 카드형 섹션 + 타임라인 + 기술 태그
- 첫 인상: "백엔드 중심인데 범위가 넓은 개발자"

디자인 원칙:

- 지나치게 화려한 애니메이션은 지양
- 한 화면에 너무 많은 문단을 넣지 않기
- 프로젝트는 결과보다 "무엇을 구현했고 어떤 맥락이었는지" 중심으로 요약
- 배경은 단색보다 미묘한 그라데이션 또는 질감 사용 가능
- 모바일 가독성을 우선 고려

## 구현 지침

초기 버전은 GitHub Pages 배포 편의성을 위해 정적 사이트 우선으로 구현한다.

- 기본 진입점은 `/index.html`
- 스타일은 초기에는 인라인 또는 단일 CSS 파일 허용
- 섹션 이동을 위한 앵커 네비게이션 허용
- 외부 빌드 도구 없이도 열리는 구조를 우선

사이트가 커지면 아래 구조로 정리할 수 있다.

- `index.html`
- `assets/styles/`
- `assets/images/`
- `assets/data/`
- `assets/scripts/`
