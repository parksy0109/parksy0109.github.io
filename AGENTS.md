# AGENTS.md

## 목적

이 저장소는 박수용의 GitHub Pages 포트폴리오를 운영하기 위한 저장소다.
기준 원본은 경력 기술서이며, 사이트에 들어가는 경력·프로젝트·기술 정보는 원본 이력서에서 확인 가능한 사실만 사용한다.

이 문서를 수정하거나 사이트를 구현하는 에이전트는 아래 기준을 반드시 따른다.

## 가장 중요한 원칙

이 포트폴리오는 "대표 사례 몇 개만 보여주는 랜딩 페이지"가 아니라, 원본 이력서에 있는 경력 항목을 빠짐없이 담는 포트폴리오여야 한다.

즉, 아래 3가지는 모두 포함되어야 한다.

1. 재직 이력 6개
2. 회사 프로젝트 8개
3. 개인/학교 프로젝트 4개

요약은 가능하지만, 항목 자체를 누락하면 안 된다.

## 디자인 원칙

이 사이트는 평범한 이력서 문서처럼 보이면 안 된다.

현재 디자인 기준 레퍼런스는 저장소 내부 `google-stitch/` 폴더다.

- `google-stitch/DESIGN.md`
- `google-stitch/code.html`
- `google-stitch/screen.png`

원하는 인상:

- 실무형 개발자 포트폴리오
- 단정하지만 강한 첫 화면
- 정보량은 많지만 읽기 쉬운 구조
- 카드와 타임라인 중심의 편집형 레이아웃
- 다크 네온 톤의 디지털 도시에 가까운 분위기

디자인 지침:

- 기본 배경은 단색보다 질감 있는 레이어드 배경 사용
- 상단 Hero는 시각적으로 강하게 구성
- 긴 경력 정보는 카드/타임라인/태그로 압축
- 모바일에서도 카드가 무너지지 않게 반응형 유지
- 밋밋한 기본 템플릿 느낌을 피한다
- `google-stitch` 레퍼런스의 좌측 레일 네비게이션, 글래스 패널, 네온 액센트, 벤토형 카드 구조를 우선 참고한다
